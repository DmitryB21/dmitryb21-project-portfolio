# Инструкция по сбросу пароля пользователя в Active Directory

## Введение
Данная инструкция предназначена для администраторов информационных систем, работающих с Microsoft Active Directory (AD). Она описывает порядок действий администратора для безопасного и корректного сброса пароля учетной записи пользователя.

## Шаги по сбросу пароля пользователя

### 1. Подключение к консоли управления Active Directory
- Откройте **Active Directory Users and Computers** через оснастку **Управление компьютером** (`MMC`).
- Выберите раздел **Пользователи**, щелкните правой кнопкой мыши и выберите пункт **Подключить новый каталог**.
- Укажите адрес домена (например, `dc=example,dc=com`) и нажмите **ОК**.

### 2. Поиск учетной записи пользователя
- Перейдите в раздел **Пользователи** и найдите учетную запись нужного пользователя.
- Если вы знаете имя пользователя, используйте фильтр поиска (**Поиск объектов**) или перейдите непосредственно к нужному подразделению.

### 3. Права и привилегии
- Убедитесь, что у вас есть права администратора домена либо соответствующие разрешения на изменение паролей пользователей.
- Для проверки прав воспользуйтесь утилитой **dsa.msc**: откройте **Свойства домена**, перейдите во вкладку **Безопасность**, убедитесь, что ваш аккаунт имеет право изменять пароль пользователя.

### 4. Сброс пароля
- Щелкните правой кнопкой мыши на нужной учетной записи и выберите **Изменить пароль**.
- Заполните поля следующим образом:
  - **Новый пароль:** введите новый пароль пользователя.
  - **Подтверждение нового пароля:** повторите ввод нового пароля.
  - **Дата истечения срока действия пароля:** оставьте значение по умолчанию (не устанавливать срок истечения).
- Нажмите **Применить** и подтвердите действие нажатием кнопки **ОК**.

### 5. Проверка сброса пароля
- После изменения пароля рекомендуется проверить работоспособность учетной записи.
- Используйте командную строку или PowerShell для выполнения команды:
  
```powershell
net use \\servername /user:"username" "newpassword"
```

или же выполните вход вручную в систему под учетной записью пользователя.

### 6. Логирование изменений
- Обязательно зафиксируйте выполненные действия в журнале событий безопасности AD.
- Рекомендуется создать документ с подробным описанием причины сброса пароля, даты и времени операции, а также указанием сотрудника, инициировавшего операцию.

## Заключение
Сброс пароля должен проводиться исключительно в случае крайней необходимости, когда пользователь забыл пароль или произошла утрата ключа восстановления. Выполнение данной процедуры требует строгого соблюдения политики информационной безопасности организации.
