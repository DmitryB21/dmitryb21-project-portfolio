# Политика резервного копирования данных

## 1. Общие положения
Настоящая политика регламентирует порядок организации и проведения регулярного резервного копирования (бекапа) данных информационных систем компании **ООО “ТехноСервис”**.

## 2. Цели политики
- Предотвращение потери критически важных данных вследствие аппаратных сбоев, ошибок программного обеспечения, вирусных атак и прочих форс-мажорных обстоятельств.
- Возможность восстановления данных в случае утраты основной базы данных.

## 3. Основные требования и обязанности
### 3.1 Ответственность за организацию резервного копирования
- Руководство ИТ-отдела несет ответственность за разработку, внедрение и контроль выполнения данной политики.
- Сотрудники подразделений обязаны своевременно сообщать администратору системы обо всех изменениях в структуре данных и приложениях.

### 3.2 Регулярность резервного копирования
Резервное копирование проводится ежедневно:
- В рабочие дни — каждый рабочий день после завершения рабочего дня до 23:00.
- В выходные и праздничные дни — один раз в неделю, предпочтительно в субботу утром.

## 4. Объекты резервного копирования
Подлежат регулярному бекапированию следующие данные:
- Операционные системы серверов и рабочих станций.
- Приложения и базы данных корпоративной информационной системы.
- Конфигурационные файлы и настройки сетевого оборудования.
- Электронная почта сотрудников.
- Документы общего пользования и личные архивы пользователей.

## 5. Способы резервного копирования
Используются следующие методы хранения резервных копий:
- Локальные жесткие диски серверов.
- Внешние USB-накопители.
- Системы облачного хранения (например, Яндекс.Диск, Google Drive).
- NAS-хранилища.

Рекомендуется использовать несколько независимых каналов хранения, чтобы обеспечить отказоустойчивость.

## 6. Процесс резервного копирования
### Шаг 1. Подготовка среды
Перед началом резервного копирования необходимо убедиться, что выполняются следующие условия:
- Все пользователи завершили работу с системой.
- Активированы необходимые службы и приложения.

### Шаг 2. Запуск процесса резервного копирования
Процесс выполняется автоматически согласно расписанию, установленному администратором ИТ-систем.

### Шаг 3. Проверка целостности резервных копий
После завершения процедуры резервного копирования производится проверка корректности созданных архивов. Для этого используется специализированный инструмент мониторинга состояния резервных копий.

### Шаг 4. Хранение резервных копий
Созданные копии хранятся в течение следующих сроков:
- Ежедневные копии — 7 дней.
- Еженедельные копии — 3 месяца.
- Годовые копии — бессрочно.

### Шаг 5. Удаление устаревших резервных копий
Удаляются копии, срок хранения которых истек. Процедура удаления должна проводиться регулярно и документироваться.

## 7. Ответственность за нарушение требований политики
Нарушение установленных правил резервного копирования влечет административную ответственность согласно локальным нормативным актам компании.

## Заключение
Соблюдение настоящей политики является обязательным условием стабильной работы и защиты корпоративных данных компании.
