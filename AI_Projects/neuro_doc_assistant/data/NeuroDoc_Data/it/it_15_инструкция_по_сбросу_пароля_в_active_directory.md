# Инструкция по сбросу пароля учётной записи в Active Directory

## Введение
Сброс пароля — распространённая задача администратора домена Active Directory (AD). Данная инструкция предназначена для сотрудников службы технической поддержки и администраторов безопасности, обеспечивающих корректное выполнение процедуры восстановления утраченного пароля пользователя.

## Шаги по восстановлению пароля через Active Directory

### 1. Подключение к консоли управления Active Directory
- Откройте командную строку Windows с правами администратора.
- Выполните команду `dsrm /connect` для подключения к локальному контроллеру домена.
- Если подключение осуществляется удалённо, используйте команду `dsrm /connect <имя_контроллера>`.

### 2. Проверка текущего состояния учетной записи
- Убедитесь, что учётная запись пользователя активна и не заблокирована.
- Для проверки выполните команду:
  
```cmd
net user имя_учетки /domain
```

### 3. Сброс пароля средствами PowerShell
- Запустите PowerShell от имени администратора.
- Используйте команду для изменения пароля:

```powershell
Set-AdAccountPassword -Identity "имя_учетки@домен.ru" -NewPassword (ConvertTo-SecureString "новыйпароль" -AsPlainText -Force)
```

**Примечание:** 
Пароль должен содержать минимум восемь символов, включать заглавные буквы, строчные буквы, цифры и специальные символы.

### 4. Изменение пароля вручную через AD Users & Computers
- Перейдите в оснастку **Active Directory пользователи и компьютеры**.
- Найдите учётную запись пользователя и дважды кликните её.
- Выберите вкладку **«Свойства»**, перейдите во вкладку **«Параметры»**.
- Нажмите кнопку **«Изменить пароль»** и введите новый пароль согласно требованиям политики паролей.

### 5. Проверьте успешность выполнения команды
- После изменения пароля убедитесь, что учётная запись успешно восстановлена.
- Команда:

```cmd
net user имя_учетки /domain
```

### 6. Отправка уведомления пользователю
- Сообщите пользователю о новом пароле и попросите сменить его сразу после входа в систему.
- Рекомендуется использовать шаблон письма, содержащий инструкции по смене пароля и рекомендации по безопасности.

### 7. Отчётность и регистрация действий
- Зарегистрируйте факт восстановления пароля в журнале событий системы и зафиксируйте действия в документации.
- Сохраните копию журнала изменений учётной записи в архиве службы поддержки.

## Заключение
Выполнение данной инструкции позволяет оперативно восстановить доступ пользователя к ресурсам компании, минимизируя риски потери рабочего времени и данные конфиденциальной информации.
