# Инструкция по сбросу пароля пользователя в Active Directory

## Введение
Данная инструкция предназначена для администраторов информационных систем организации и описывает порядок действий по сбросу пароля учетной записи пользователя в среде Microsoft Active Directory (AD).

## Требования к выполнению процедуры
- Администратор должен обладать правами администратора домена (Domain Admins), локального администратора (Local Administrators) или привилегиями группы Enterprise Admins.
- Доступ к консоли управления Active Directory Users and Computers (ADUC) или командной строке Windows PowerShell.

## Шаги по сбросу пароля пользователя

### 1. Открытие консоли Active Directory Users and Computers
Откройте оснастку Active Directory Users and Computers через меню «Пуск» → «Администрирование» → «Active Directory пользователей и компьютеров».

### 2. Поиск учетной записи пользователя
Найдите нужную учетную запись пользователя, используя фильтр поиска или вручную просмотрев каталог.

### 3. Правый клик на учетной записи и выбор опции «Сброс пароля»
Правым кликом мыши выберите учетную запись пользователя и нажмите «Свойства», далее перейдите во вкладку «Пароль» и выберите опцию «Сброс пароля».

### 4. Подтверждение сброса пароля
Подтвердите операцию сброса пароля, введя пароль текущего администратора домена и подтвердив действие.

### 5. Установка нового пароля
После успешного сброса системы автоматически предложит установить новый пароль. Укажите новый пароль и подтвердите его ввод дважды.

### 6. Проверка установки пароля
Убедитесь, что система правильно установила новый пароль, нажав кнопку «Готово». После завершения операции вы получите уведомление о сбросе пароля.

### Альтернативный способ сброса пароля через PowerShell
Для автоматизированного выполнения данной процедуры рекомендуется использовать командлет PowerShell:

```powershell
Set-ADAccountPassword -Identity <имя_учётной_записи> -NewPassword (ConvertTo-SecureString "<новый_пароль>" -AsPlainText -Force)
```

Пример команды:
```powershell
Set-ADAccountPassword -Identity "Иван.Иванов" -NewPassword (ConvertTo-SecureString "newpassword123!" -AsPlainText -Force)
```

### Дополнительная информация
При выполнении данной процедуры важно соблюдать следующие рекомендации:
- Новый пароль должен соответствовать политике безопасности паролей компании.
- Рекомендуется уведомить пользователя о новом пароле и напомнить о соблюдении правил информационной безопасности.
- Обязательно зафиксируйте факт сброса пароля в журнале событий AD и документации по инцидентам.

## Заключение
Выполнение данной инструкции позволит быстро и эффективно восстановить доступ пользователя к корпоративному ресурсу после утраты пароля.
