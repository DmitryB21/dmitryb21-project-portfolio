## 1. Общие сведения

### Назначение инструкции
Настоящая инструкция предназначена для сотрудников службы технической поддержки компании, администраторов информационных систем и специалистов ИТ-отдела, выполняющих функции управления пользователями корпоративной сети. Она описывает порядок выполнения процедуры сброса пароля пользователя в среде Microsoft Active Directory.

### Требования безопасности
Перед выполнением данной процедуры необходимо убедиться, что выполняются следующие условия:
- Доступ к управлению AD осуществляется исключительно авторизованным пользователям.
- Все действия фиксируются в журнале событий системы.
- Пароли пользователей хранятся в зашифрованном виде.

## 2. Подготовка к выполнению процедуры

### Подготовка учетной записи администратора домена
Для успешного выполнения процедуры необходим доступ от учетной записи администратора домена, обладающего правами локального администратора или привилегиями управления пользователями.

### Проверка прав доступа
Убедитесь, что ваша учетная запись имеет права администратора домена либо включена в группу «Администраторы домена», «Пользователи группы администраторов» или аналогичную административную роль.

---

## 3. Процедура сброса пароля

### Шаг 1. Вход в консоль управления Active Directory (AD)
1. Войдите в систему Windows Server под учетной записью администратора домена.
2. Перейдите в раздел «Панель управления → Администрирование → Управление компьютером → Службы каталогов и Active Directory → Active Directory Users and Computers».

### Шаг 2. Открытие окна свойств пользователя
1. В открывшемся окне выберите нужного пользователя.
2. Щелкните правой кнопкой мыши и выберите пункт «Свойства».

### Шаг 3. Выбор опции изменения пароля
1. Во вкладке «Пароль» нажмите ссылку «Изменить пароль…».
2. Выберите опцию «Сбросить пароль учетной записи».

### Шаг 4. Установка нового пароля
1. Установите новый пароль согласно политике организации.
2. Повторите ввод установленного пароля во второй строке подтверждения.

### Шаг 5. Завершение операции и проверка результата
1. Нажмите «ОК» для завершения процедуры.
2. Проверьте успешность сброса пароля через просмотр журнала событий.

---

## 4. Дополнительные меры безопасности

После сброса пароля рекомендуется уведомить пользователя о произошедших изменениях, отправить письмо с новым паролем и попросить сменить его сразу после входа в систему.

Записывайте все выполненные действия в журнал событий для последующего аудита и контроля над действиями администратора.
