# Инструкция по работе с инструментами Atlassian — Jira и Confluence

## Введение
Документ описывает порядок работы с системами управления проектами **Jira** и ведения документации **Confluence**, используемыми в компании. Прочитав инструкцию, сотрудники смогут эффективно использовать данные инструменты для организации рабочего процесса, постановки задач, отслеживания прогресса проектов и документирования процессов.

---

## Часть I. Работа с системой Jira

### 1. Регистрация в Jira
- Перейдите на официальный сайт [jira.atlassian.com](https://jira.atlassian.com).
- Нажмите кнопку «Зарегистрироваться», выберите вариант регистрации через электронную почту (рекомендуемый способ).
- Заполните регистрационную форму, указав корректные контактные данные и пароль.
  
### 2. Вход в систему Jira
- Откройте браузер и перейдите на корпоративный портал Jira.
- Авторизуйтесь введением зарегистрированного e-mail и пароля.

### 3. Создание новой задачи (Issue)
- Выберите проект, где будет создана задача.
- Кликните «Создать новую задачу» («Create issue»).
- Укажите тип задачи (Bug, Task, Story и др.), добавьте описание проблемы/задачи, укажите приоритет и срок выполнения.
- Сохраните изменения нажатием кнопки «Создать» («Create»).

### 4. Просмотр и фильтрация задач
- Используйте встроенные фильтры для поиска нужных задач по статусу, автору, типу или другим параметрам.
- Применяйте сортировку по дате добавления, приоритета или статуса.

### 5. Отслеживание прогресса проекта
- Следите за изменениями состояния задач в колонке «История изменений» («Change log»).
- Оценивайте прогресс команды по диаграмме Burndown chart, доступной в панели проекта.

---

## Часть II. Использование системы Confluence

### 6. Регистрация в Confluence
- Зайдите на страницу регистрации [confluence.atlassian.com](https://confluence.atlassian.com).
- Зарегистрируйтесь, используя корпоративную почту и надежный пароль.

### 7. Вход в систему Confluence
- Войдите в Confluence через корпоративный портал.

### 8. Создание страницы документа
- Выберите раздел проекта, куда нужно добавить документацию.
- Создайте новую страницу, выбрав соответствующий шаблон (например, руководство пользователя, инструкция, чек-лист).
- Напишите заголовок, вставьте необходимые таблицы, графики и иллюстрации.

### 9. Управление версиями документов
- Для каждого обновления создавайте новые версии страниц.
- Назначайте ответственное лицо за обновление содержания.
- Оставляйте комментарии и заметки по каждому изменению.

### 10. Совместная работа над документом
- Предоставляйте доступ коллегам для редактирования страниц.
- Используйте режим совместной работы («Edit together») для одновременной правки текста несколькими пользователями.

---

## Заключение
Использование инструментов Jira и Confluence позволяет эффективно управлять процессами разработки, улучшения качества продукта и повышения прозрачности взаимодействия внутри команды. Соблюдение данной инструкции поможет избежать ошибок и повысить эффективность командной работы.
