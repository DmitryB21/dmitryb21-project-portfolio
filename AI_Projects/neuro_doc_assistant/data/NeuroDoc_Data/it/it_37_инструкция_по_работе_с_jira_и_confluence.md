# Инструкция по работе с системами Jira и Confluence

## Введение
Системы управления проектами **Jira** и **Confluence** от компании Atlassian широко используются в российских компаниях для организации работы команд разработчиков, менеджеров проектов и тестировщиков. Данная инструкция предназначена для сотрудников, работающих с этими инструментами ежедневно.

## Шаг 1. Регистрация и авторизация в системах

### 1.1 Авторизация через корпоративный портал (SberID)
- Перейдите на официальный сайт компании (например, [corp.sber.ru](https://corp.sber.ru)).
- Войдите в систему используя ваш личный аккаунт **SberID**.
- После успешной аутентификации перейдите в раздел корпоративных сервисов.

### 1.2 Вход в системы Jira и Confluence
- Откройте браузер и зайдите на сайты:
  - **Jira**: https://jira.corp.sber.ru/
  - **Confluence**: https://confluence.corp.sber.ru/
- Используйте имя пользователя и пароль, выданные вам администратором портала.

## Шаг 2. Работа с системой Jira

### 2.1 Создание нового проекта
- Перейдите в меню верхнего уровня → выберите раздел «Проект» → нажмите «Создать проект».
- Выберите шаблон проекта, подходящий вашей команде (например, Agile, Scrum).
- Заполните необходимые поля: название, описание, тип команды и укажите участников.

### 2.2 Управление задачами и багами
- Для отслеживания задач используйте доску Kanban или спринт-планирование.
- Создавайте новые задачи, указывая тип задачи («Bug», «Task», «Story»), приоритет и сроки выполнения.
- Отслеживайте статус задачи, взаимодействуйте с командой через комментарии и обсуждения.

### 2.3 Просмотр отчетов и метрик
- Зайдите в раздел «Отчеты» → выберите интересующий вас отчет (например, Burndown chart, Velocity chart).
- Анализируйте данные и принимайте решения на основе полученных показателей эффективности.

## Шаг 3. Использование системы Confluence

### 3.1 Создание новой страницы
- Нажмите значок «+» в правом верхнем углу → выберите опцию «Новая страница».
- Укажите название страницы и добавьте необходимую информацию (текст, таблицы, диаграммы).

### 3.2 Организация структуры знаний
- Используйте структуру страниц и разделов для систематизации документации.
- Добавляйте гиперссылки между страницами для удобства навигации.

### 3.3 Обновление и редактирование документов
- Чтобы обновить существующую страницу, откройте её и внесите изменения.
- Обязательно сохраняйте документ после внесения правок.

## Заключение
Знание принципов работы с Jira и Confluence позволяет эффективно управлять процессами разработки программного обеспечения, организовывать совместную работу и улучшать коммуникацию внутри команды. Соблюдение данной инструкции поможет повысить продуктивность и качество вашего труда.
